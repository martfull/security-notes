# Тема 9. Безпека комп'ютерних мереж

## 1. Модель мережевої безпеки

Передача повідомлень через мережу вимагає двох компонентів захисту:
- **Перетворення** — шифрування повідомлення
- **Секретна інформація** — ключ, відомий лише довіреним сторонам

### Чотири завдання при розробці сервісу безпеки:
1. Розробка алгоритму захисту
2. Генерація секретної інформації (ключів)
3. Розподіл ключів
4. Вибір протоколу передачі

---

## 2. Загрози мережевій безпеці

| Тип загрози | Приклад |
|-------------|---------|
| **Перехоплення** | Підслуховування трафіку, знімання паролів |
| **Підробка** | Підміна IP-адреси (IP Spoofing) |
| **Модифікація** | Зміна пакетів у Transit |
| **Відмова в обслуговуванні** | DDoS-атаки |
| **Несанкціонований доступ** | Злом через вразливості |

---

## 3. Міжмережеві екрани (Firewall)

**Firewall** — система або пристрій, що контролює мережевий трафік між зонами різного рівня довіри.

### Типи файрволів:

| Тип | Рівень OSI | Принцип |
|-----|------------|---------|
| **Пакетний фільтр** | 3–4 | Фільтрація за IP, порт, протокол |
| **Stateful Inspection** | 3–4 | Відстежує стан з'єднань |
| **Проксі-файрвол** | 7 | Аналіз вмісту прикладного рівня |
| **NGFW** | 3–7 | Комбінований, з DPI та IPS |

### Принципи файрволу:
- **Whitelist** — дозволити лише необхідне
- **Blacklist** — заблокувати відоме шкідливе
- **Default Deny** — все заблоковано за замовчуванням

---

## 4. DMZ — Демілітаризована зона

```
Internet → Firewall 1 → [DMZ: Web/Mail сервери] → Firewall 2 → Internal Network
```

- Публічно доступні сервери розміщуються в DMZ
- DMZ ізольована від внутрішньої мережі
- Навіть при компрометації DMZ — внутрішня мережа захищена

---

## 5. VPN — Віртуальна приватна мережа

**VPN** — захищений тунель через публічну мережу.

### Типи VPN:
| Тип | Застосування |
|-----|-------------|
| **Site-to-Site VPN** | З'єднання двох офісів |
| **Remote Access VPN** | Підключення віддалених працівників |
| **Client-to-Server** | Доступ до конкретного сервера |

### Протоколи VPN:
- **IPSec** — протокол рівня IP, забезпечує конфіденційність та цілісність
- **SSL/TLS VPN** — через HTTPS, простий у розгортанні
- **OpenVPN** — відкритий, гнучкий
- **WireGuard** — сучасний, швидкий

---

## 6. SSL/TLS — Захист веб-трафіку

**TLS** (Transport Layer Security) — криптографічний протокол для захисту каналу зв'язку.

### TLS Handshake:
```
1. Client Hello (версія, cipher suites, random)
2. Server Hello (обраний cipher, random)
3. Server Certificate (X.509)
4. Key Exchange (Diffie-Hellman або RSA)
5. Finished (все зашифровано)
```

### Версії:
- SSL 3.0, TLS 1.0, 1.1 — **застарілі, небезпечні**
- **TLS 1.2** — широко використовується
- **TLS 1.3** — сучасний стандарт, швидший

---

## 7. Протоколи безпеки мереж

| Протокол | Рівень | Функція |
|----------|--------|---------|
| **IPSec** | Мережевий (3) | Шифрування IP-пакетів |
| **TLS/SSL** | Транспортний (4) | Захищений канал |
| **SSH** | Прикладний (7) | Захищений термінал, тунелювання |
| **HTTPS** | Прикладний (7) | HTTP поверх TLS |
| **SFTP** | Прикладний (7) | Захищена передача файлів |

---

## 8. DNS Security

### DNS-атаки:
- **DNS Spoofing** — підміна DNS-відповідей
- **DNS Amplification** — DDoS через DNS
- **DNS Hijacking** — перенаправлення запитів

### Захист:
- **DNSSEC** — цифровий підпис DNS-записів
- **DoH** (DNS over HTTPS) — шифрування DNS-запитів
- **DoT** (DNS over TLS) — шифрування DNS через TLS

---

## 9. Сегментація мережі

```
Network Zones:
┌──────────────────────────────────────────┐
│  Internet (Untrusted)                    │
├──────────────────────────────────────────┤
│  DMZ (Semi-trusted): Web, Mail, DNS      │
├──────────────────────────────────────────┤
│  Corporate LAN (Trusted)                 │
├──────────────────────────────────────────┤
│  Management Network (Highly Trusted)     │
└──────────────────────────────────────────┘
```

**VLAN** — логічна сегментація мережі для ізоляції трафіку.

| Термін | Визначення |

| VPN | Захищений тунель через публічну мережу |
