# Тема 8. Безпека операційних систем

## 1. Рівні безпеки ОС

Системи мають ієрархічну архітектуру захисту:

```
┌─────────────────────────┐
│  Застосунки користувача │ ← Найвразливіший рівень
├─────────────────────────┤
│  ОС (ядро, API, служби) │
├─────────────────────────┤
│  BIOS / UEFI / Firmware │
├─────────────────────────┤
│  Апаратне забезпечення  │ ← Основа
└─────────────────────────┘
```

Кожен рівень потребує захисту. Атаки знизу компрометують вищі рівні.

---

## 2. Базові заходи захисту ОС (DSD Top 4)

Австралійське управління оборонних сигналів встановило, що **4 заходи** запобігають понад **70% цільових атак**:

1. **Оновлення ОС і програм** — автоматичне оновлення патчів
2. **Оновлення сторонніх програм** — Java, Flash, PDF-читачі
3. **Обмеження прав адміністратора** — принцип мінімального привілею
4. **Whitelist застосунків** — дозволяти лише відомі програми

---

## 3. Посилення безпеки системи (Hardening)

### Мінімалізація:
- Видалення непотрібних служб, протоколів, програм
- Відключення невикористовуваних мережевих портів
- Видалення стандартних облікових записів

### Конфігурація:
- Налаштування парольних політик
- Обмеження привілеїв облікових записів
- Захист BIOS/UEFI паролем

### Патч-менеджмент:
- Регулярне встановлення оновлень безпеки
- Тестування патчів перед розгортанням
- Використання WSUS/SCCM для централізованого управління

---

## 4. Управління обліковими записами

### Типи облікових записів:
| Тип | Рівень привілеїв | Використання |
|-----|-----------------|--------------|
| Адміністратор | Повний | Лише для адміністрування |
| Звичайний | Обмежений | Повсякденна робота |
| Служби | Мінімальний | Запуск системних служб |
| Гість | Дуже обмежений | Тимчасовий доступ |

### Найкращі практики:
- Не використовувати адміністраторський обліковий запис для роботи
- Відключити обліковий запис Guest
- Використовувати UAC (User Account Control)
- Встановлювати складні паролі + MFA

---

## 5. Файлова система та дозволи

### Linux (Unix) — моделі дозволів:
```
-rwxr-xr-- 1 user group size date file
 ↑↑↑↑↑↑↑↑↑
 │└──┘└──┘└──┘
 │ owner group others
 │
 └─ тип файлу (- файл, d директорія)

r = читання (4)
w = запис (2)
x = виконання (1)
```

### Windows — NTFS дозволи:
- Full Control, Modify, Read & Execute, Read, Write
- Наслідуються від батьківських папок
- Явна заборона має пріоритет над дозволом

## 6. Журналювання та аудит

### Що фіксувати:
- Успішні та невдалі спроби входу
- Зміни привілеїв облікових записів
- Зміни системних файлів та конфігурацій
- Запуск і зупинка служб
- Мережеві підключення

### Windows Event Log:
- **Журнал безпеки** — автентифікація, авторизація
- **Журнал системи** — системні події
- **Журнал застосунків** — події програм

### Linux syslog:
- `/var/log/auth.log` — автентифікація
- `/var/log/syslog` — системні події
- `/var/log/kern.log` — події ядра

---

## 7. Захист від шкідливого ПЗ на рівні ОС

- **AppLocker / Software Restriction Policies** — whitelist застосунків
- **Windows Defender / ClamAV** — антивірусний захист
- **SELinux / AppArmor** — мандатний контроль доступу в Linux
- **Secure Boot** — верифікація завантажувача через UEFI
- **TPM (Trusted Platform Module)** — апаратний захист ключів шифрування

---

## 8. Шифрування дисків

| Рішення | ОС | Опис |
|---------|----|------|
| **BitLocker** | Windows | Шифрування томів AES |
| **FileVault** | macOS | Шифрування диску |
| **LUKS/dm-crypt** | Linux | Повнодискове шифрування |
| **VeraCrypt** | Крос-платформне | Шифровані контейнери |

## Ключові терміни

| Термін | Визначення |
|--------|------------|
| Hardening | Посилення безпеки ОС шляхом мінімалізації та конфігурації |
| UAC | User Account Control — контроль облікових записів Windows |
| SELinux | Security-Enhanced Linux — MAC для Linux |
| TPM | Trusted Platform Module — апаратний модуль безпеки |
| Whitelist | Список дозволених програм |
