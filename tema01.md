# Тема 1. Основні поняття інформаційної безпеки (CIA, ризик, загрози)

## 1. Визначення інформаційної безпеки

**Інформаційна безпека (InfoSec)** — стан захищеності інформаційних ресурсів та систем від негативних дій, що можуть завдати збитків власникам інформації, державі або суспільству.

**Комп'ютерна безпека** — захист автоматизованої інформаційної системи з метою збереження конфіденційності, цілісності та доступності ресурсів.

---

## 2. CIA-тріада

| Властивість | Опис |
|-------------|------|
| **Confidentiality (Конфіденційність)** | Інформація доступна лише авторизованим особам. Включає конфіденційність даних і приватність. |
| **Integrity (Цілісність)** | Інформація змінюється лише авторизованим чином. Включає цілісність даних і цілісність системи. |
| **Availability (Доступність)** | Система функціонує своєчасно і не відмовляє в обслуговуванні авторизованим користувачам. |

Додаткові властивості:
- **Автентичність** — підтвердження достовірності сторін і даних
- **Підзвітність** — можливість відстежити дії суб'єкта до конкретної особи

---

## 3. Загрози безпеці

**Загроза** — потенційна можливість порушення безпеки, яка існує при наявності обставин, здібностей, дій або подій, що можуть призвести до порушення безпеки.

**Вразливість** — недолік або слабкість у проектуванні, реалізації або управлінні системою, яка може бути використана для порушення безпеки.

**Атака** — цілеспрямована дія з використанням вразливості для порушення безпеки.

### Типи атак (за RFC 2828)

| Тип | Опис | Приклад |
|-----|------|---------|
| **Пасивна** | Перехоплення без зміни даних | Підслуховування, аналіз трафіку |
| **Активна** | Зміна або підробка даних | Маскарад, відтворення, модифікація, DoS |

### Пасивні атаки
- **Підслуховування (eavesdropping)** — перехоплення конфіденційних повідомлень
- **Аналіз трафіку** — визначення характеру зв'язку навіть без розкодування

### Активні атаки
- **Маскарад (masquerade)** — підробка ідентичності
- **Відтворення (replay)** — повторне надсилання перехопленого пакету
- **Модифікація повідомлення** — зміна частини легітимного повідомлення
- **Відмова в обслуговуванні (DoS)** — блокування нормального використання сервісу

---

## 4. Ризик

**Ризик** — функція від ймовірності настання загрози та її наслідків для організації.

```
Ризик = Загроза × Вразливість × Вартість активу
```

### Методи управління ризиком:
- **Прийняття** — взяти ризик на себе
- **Зниження** — впровадити захисні заходи
- **Передача** — страхування або аутсорсинг
- **Уникнення** — відмова від ризикованої діяльності

---

## 5. Сервіси та механізми безпеки (X.800)

### Основні сервіси безпеки:
| Сервіс | Опис |
|--------|------|
| Автентифікація | Підтвердження ідентичності суб'єкта |
| Контроль доступу | Запобігання несанкціонованому використанню ресурсів |
| Конфіденційність даних | Захист від несанкціонованого розкриття |
| Цілісність даних | Гарантія незмінності даних |
| Невідмовність | Неможливість заперечити відправлення/отримання |

### Механізми безпеки:
- Шифрування
- Цифровий підпис
- Контроль доступу
- Цілісність даних
- Обмін автентифікаційними даними
- Набивка трафіку
- Управління маршрутизацією
- Нотаріальне засвідчення

---

## 6. Модель X.800 та OSI

Сервіси безпеки розподілені по рівнях моделі OSI:
- **Рівень 1 (Фізичний)**: конфіденційність трафіку
- **Рівень 3 (Мережевий)**: автентифікація, шифрування
- **Рівень 4 (Транспортний)**: автентифікація, шифрування
- **Рівень 7 (Прикладний)**: всі сервіси безпеки

## Ключові терміни

| Термін | Визначення |
|--------|------------|
| CIA | Confidentiality, Integrity, Availability — три основні властивості |
| Загроза | Потенційна можливість порушення безпеки |
| Вразливість | Слабкість системи, яку можна використати |
| Ризик | Ймовірність збитку від реалізації загрози |
| Атака | Навмисна дія проти безпеки системи |
