# Тема 7. Функції хешування та цифровий підпис

## 1. Функції хешування

**Криптографічна хеш-функція** — перетворює вхідне повідомлення довільного розміру у вихід фіксованого розміру (хеш / дайджест).

```
h = H(M)
```

### Властивості хеш-функції:
| Властивість | Зміст |
|-------------|-------|
| **Стійкість до першого прообразу** | За хешем H(x) неможливо знайти x |
| **Стійкість до другого прообразу** | За x і H(x) неможливо знайти y ≠ x, де H(y) = H(x) |
| **Стійкість до колізій** | Неможливо знайти x і y, де H(x) = H(y) |
| **Лавинний ефект** | Мала зміна входу → значна зміна хешу |

### Основні алгоритми:
| Алгоритм | Розмір хешу | Стан |
|----------|-------------|------|
| MD5 | 128 біт | Небезпечний (знайдені колізії) |
| SHA-1 | 160 біт | Небезпечний (знайдені колізії) |
| SHA-256 | 256 біт | Безпечний, рекомендований |
| SHA-384 | 384 біт | Безпечний |
| SHA-512 | 512 біт | Безпечний |
| SHA-3 | 224–512 біт | Сучасний стандарт |

### Застосування хеш-функцій:
- Перевірка цілісності файлів
- Зберігання паролів (з сіллю)
- Цифровий підпис
- MAC (код автентифікації)
- Блокчейн

## 2. Цифровий підпис

**Цифровий підпис** — криптографічна схема, що підтверджує:
- **Автентичність** — повідомлення дійсно відправлено вказаним автором
- **Цілісність** — повідомлення не було змінено
- **Невідмовність** — автор не може заперечити відправлення

### Схема роботи:
```
Підписання:
  1. Обчислити хеш: h = H(M)
  2. Підписати хеш: S = Sign(KR_A, h)
  3. Відправити M + S

Перевірка:
  1. Отримати M + S
  2. Обчислити h' = H(M)
  3. Відновити h = Verify(KU_A, S)
  4. Перевірити: h' == h ?
```

### Алгоритми цифрового підпису:
- **RSA-PSS** — RSA з заповненням
- **DSA** (Digital Signature Algorithm) — стандарт FIPS
- **ECDSA** (Elliptic Curve DSA) — на еліптичних кривих, ефективніший
- **EdDSA** (Edwards-curve DSA) — сучасний, дуже швидкий (Ed25519)

## 3. Управління ключами

### Проблема розподілу відкритих ключів

Методи розповсюдження відкритих ключів:

| Метод | Безпека | Недоліки |
|-------|---------|----------|
| **Публічне оголошення** | Низька | Будь-хто може підробити ключ |
| **Публічний каталог** | Середня | Каталог може бути скомпрометований |
| **Авторитетне джерело** | Висока | Потребує постійного зв'язку з центром |
| **Сертифікати** | Висока | Потребує PKI-інфраструктури |

## 4. PKI — Інфраструктура відкритих ключів

**PKI** (Public Key Infrastructure) — система управління цифровими сертифікатами.

### Компоненти PKI:
- **CA (Certificate Authority)** — засвідчувальний центр, видає сертифікати
- **RA (Registration Authority)** — перевірка особи перед видачею сертифікату
- **CRL (Certificate Revocation List)** — список відкликаних сертифікатів
- **OCSP** — онлайн-перевірка статусу сертифікату

### Сертифікат X.509:
```
Сертифікат містить:
- Ім'я власника
- Відкритий ключ власника
- Термін дії
- Підпис CA
- Серійний номер
- Алгоритм підпису
```

### Ланцюжок довіри:
```
Root CA (довірений)
    └── Intermediate CA
            └── End-entity Certificate (сервер/користувач)
```

## 5. Схема Діффі–Хеллмана (детально)

Безпечний обмін ключами через незахищений канал.

```
Параметри: p (велике просте число), g (генератор)

1. Аліса: a (секрет), A = g^a mod p (відкрите)
2. Боб:   b (секрет), B = g^b mod p (відкрите)
3. Обмін: Аліса отримує B, Боб отримує A
4. Аліса: S = B^a mod p = g^(ab) mod p
5. Боб:   S = A^b mod p = g^(ab) mod p
→ Спільний секрет S однаковий, але не передавався!
```

**Вразливість**: атака «людина посередині» (MITM) — треба автентифікувати відкриті ключі.

## 6. HMAC — Код автентифікації на основі хешу

```
HMAC(K, M) = H((K' XOR opad) || H((K' XOR ipad) || M))
```

- Використовує симетричний ключ K та хеш-функцію H
- Захищає від підробки повідомлень
- Не забезпечує невідмовність (обидві сторони знають ключ)
