# Тема 11. Системи виявлення та запобігання вторгненням (IDS/IPS)

## 1. Підходи до боротьби зі шкідливим ПЗ

### Стратегія захисту:
1. **Запобігання** — ідеальна мета, але практично недосяжна повністю
2. **Виявлення** — після зараження, визначити що воно відбулося
3. **Ідентифікація** — визначити конкретне шкідливе ПЗ
4. **Видалення** — усунути всі сліди зараження

### 4 основні елементи запобігання (NIST):
| Елемент | Зміст |
|---------|-------|
| **Політика** | Правила щодо запобігання шкідливому ПЗ |
| **Обізнаність** | Навчання користувачів |
| **Зменшення вразливостей** | Оновлення, патчі |
| **Зменшення загроз** | Контролі доступу |

## 2. Антивірусний захист

### Методи виявлення шкідливого ПЗ:

| Метод | Принцип | Переваги | Недоліки |
|-------|---------|----------|----------|
| **Сигнатурний** | Порівняння з відомими зразками | Точний, швидкий | Не виявляє нові загрози |
| **Евристичний** | Аналіз поведінки та структури | Виявляє нові варіанти | Хибні спрацьовування |
| **Поведінковий** | Моніторинг дій у реальному часі | Виявляє невідомі загрози | Ресурсомісткий |
| **Sandbox** | Запуск у ізольованому середовищі | Безпечно | Повільний |
| **Machine Learning** | AI-класифікація | Адаптивний | Потребує навчання |

## 3. IDS — Системи виявлення вторгнень

**IDS** (Intrusion Detection System) — моніторить мережу або систему та виявляє підозрілу активність.

### Типи IDS:

| Тип | Розміщення | Що моніторить |
|-----|------------|---------------|
| **NIDS** (Network IDS) | Мережа | Мережевий трафік |
| **HIDS** (Host IDS) | Хост | Файли, процеси, журнали |
| **WIDS** (Wireless IDS) | Wi-Fi | Бездротовий трафік |

### Методи виявлення IDS:

**1. Виявлення на основі сигнатур**
- Порівняння з базою відомих атак
- Ефективно для відомих загроз
- Не виявляє zero-day атаки

**2. Виявлення аномалій**
- Порівняння з нормальним профілем поведінки
- Виявляє нові типи атак
- Вищий рівень хибних спрацьовувань

**3. Stateful Protocol Analysis**
- Аналіз відповідності протоколу специфікації
- Виявляє відхилення від нормального використання протоколу

## 4. IPS — Системи запобігання вторгненням

**IPS** (Intrusion Prevention System) — IDS + здатність активно блокувати загрози.

```
IDS: Виявляє → Повідомляє адміністратора
IPS: Виявляє → Автоматично блокує + Повідомляє
```

### Розміщення IPS:
- **Inline** — весь трафік проходить через IPS (може блокувати)
- **Out-of-band** — копія трафіку (лише моніторинг)

### Реакції IPS:
- Блокування пакетів/з'єднань
- Скидання TCP-сесії (RST пакет)
- Блокування IP-адреси
- Реконфігурація файрволу

## 5. SIEM — Управління подіями безпеки

**SIEM** (Security Information and Event Management) — збір, кореляція та аналіз подій безпеки з усіх джерел.

```
Джерела → [Колектори] → SIEM → [Аналіз + Кореляція] → Алерти
  │
  ├── Firewall logs
  ├── IDS/IPS logs
  ├── Server logs
  ├── Application logs
  └── Network flows
```

### Функції SIEM:
- Централізоване зберігання логів
- Кореляція подій з різних джерел
- Виявлення аномалій
- Автоматичні сповіщення
- Відповідність вимогам (compliance)
- Форензика та розслідування інцидентів

### Відомі SIEM-системи:
- Splunk
- IBM QRadar
- Microsoft Sentinel
- Elastic SIEM
- AlienVault OSSIM (open source)

## 6. EDR — Виявлення та реагування на ендпоінтах

**EDR** (Endpoint Detection and Response) — сучасний розвиток антивірусного захисту.

### Відмінності від традиційного антивірусу:
| Характеристика | Антивірус | EDR |
|----------------|-----------|-----|
| Фокус | Відомі загрози | Поведінка та аномалії |
| Реагування | Автоматичне | Автоматичне + Ручне |
| Форензика | Обмежена | Детальна |
| Видимість | Обмежена | Повна |

---

## 7. Honeypot / Honeynet

**Honeypot** — навмисно вразлива система для залучення атакуючих і дослідження їх дій.

**Мета:**
- Відволікти атакуючих від реальних систем
- Вивчити методи атак
- Зібрати IOC (Indicators of Compromise)

---

## 8. Вимоги до ефективних засобів захисту

| Вимога | Зміст |
|--------|-------|
| **Загальність** | Боротьба з широким спектром атак |
| **Своєчасність** | Швидка реакція для обмеження шкоди |
| **Стійкість** | Стійкість до методів ухилення |
| **Мінімальні витрати** | Не деградує продуктивність системи |
| **Прозорість** | Не потребує зміни існуючих систем |
| **Глобальне охоплення** | Може діяти на різних рівнях |
